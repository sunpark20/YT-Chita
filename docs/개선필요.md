# 개선 필요 사항

## 1. 디스크 용량 사전 체크 (우선순위: 높음)

다운로드 시작 전 남은 디스크 용량 체크가 없음. 대량 다운로드 시 디스크 가득 차면 파일 손상 가능.

- `shutil.disk_usage()`로 남은 용량 확인하는 API 추가
- 프론트엔드에서 다운로드 전 용량 부족 시 경고 모달 표시

**관련 파일:** `src/api/routes.py`, `src/frontend/js/app.js`

## 2. 아키텍처 문서 정정 (우선순위: 중간)

`docs/architecture.md`에 SSE(Server-Sent Events)라고 적혀 있지만 실제 구현은 1초 간격 HTTP 폴링임.

- `"Fetch API / SSE"` → `"Fetch API / Polling"` 수정
- `"SSE 진행률 스트리밍"` → `"1초 폴링 (GET /download/progress/{video_id})"` 수정
- 2-worker 동시 다운로드 패턴 반영 (app.js:724 `Promise.all([worker(), worker()])`)

**관련 파일:** `docs/architecture.md`

## 3. 다운로드 진행률 맵 thread safety (우선순위: 중간)

2-worker 동시 다운로드 시 `_progress_map` 등 dict에 동시 쓰기 가능. CPython GIL이 보호해주지만 명시적 Lock이 안전함.

- `downloader.py`에 `threading.Lock()` 추가
- `progress_hook`, `postprocessor_hook`, `get_progress()`에 Lock 적용

**관련 파일:** `src/services/downloader.py`

## 4. 미사용 코드 정리 (우선순위: 낮음)

- `routes.py:10` — `StreamingResponse` import 사용하지 않음
- `routes.py:753` — `/download/extract` endpoint 프론트엔드에서 미사용

**관련 파일:** `src/api/routes.py`
