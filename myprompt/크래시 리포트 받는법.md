# 크래시 리포트 시스템

## 동작 구조

```
앱 크래시 발생
    ↓
crash_reporter.py 가 시스템 정보 수집
    ↓
Google Apps Script 웹훅에 JSON POST
    ↓
sun.park20@gmail.com 으로 이메일 발송
```

## 수집 정보

- 앱 버전, OS, 아키텍처, Python 버전
- 에러 타입 + traceback 전문
- 최근 로그 50줄 (당일 로그 파일에서 추출)

## 리포트가 발송되는 시점

1. **예상치 못한 예외** — `sys.excepthook`에서 자동 호출
2. **main() 최외곽 예외** — `__main__`의 except 블록에서 호출
3. **서버 시작 실패** — FastAPI 서버가 10초 내 응답 없을 때

## 특징

- 별도 스레드에서 전송 (앱 종료를 최대 5초만 지연)
- 같은 세션에서 동일 에러 중복 전송 안 함
- 표준 라이브러리만 사용 (추가 의존성 없음)
- 웹훅 전송 실패해도 앱 동작에 영향 없음

## 관련 파일

| 파일 | 역할 |
|------|------|
| `src/utils/crash_reporter.py` | 크래시 수집 + 웹훅 전송 |
| `src/main.py` | crash_reporter 호출 지점 3곳 |
