# URL 감지 규칙 (detectUrlType 순서)

1. `/shorts/` 포함 → **개별 동영상** (`video`)
2. `playlist?list=` 또는 `&list=` 포함 → **재생목록** (`playlist`)
3. `/playlists`로 끝남 → **채널의 모든 재생목록** (`channel_playlists`)
4. `watch?v=`, `youtu.be/`, 11자리 ID → **개별 동영상** (`video`)
5. 위 모두 아님 → **채널** (`channel`)

---

# 헷갈리는 포인트

- `watch?v=XXX&list=YYY` → 규칙2에 먼저 걸림 → **재생목록 전체** 다운로드 (개별 동영상 아님)
- Shorts URL → 채널이 아니라 **개별 동영상**으로 처리
- `@핸들` 입력 → 백엔드에서 `https://www.youtube.com/@핸들`로 자동 변환 → 채널
- 채널 분석일 때만 `include_shorts` (3분 미만 필터) 옵션 전송됨. 다른 타입에선 무시

---

# 키보드 흐름

- URL 입력 → Enter → 화질 드롭다운 포커스
- 화질 변경 시 → 분석 버튼 포커스
- 화질에서 Enter (변경 없어도) → 바로 분석 시작

---

# 다운로드 저장 경로 (~/Downloads/YouTubeDownloader/)

- 채널: `채널명/All Videos/`
- 재생목록: `채널명/재생목록명/`
- 채널 플레이리스트: `채널명/각 재생목록명/` (동명 재생목록은 `이름/이름 (1)` 구조)
- 개별 동영상: `채널명/All Videos/`

---

# 중복/스킵 처리

- `filter_already_downloaded`: 폴더에 이미 파일 있으면 목록에서 제외
- `download_archive`: 다운로드 완료 기록. 멤버십 전용도 기록하여 재시도 방지
- 두 가지 모두 통과해야 실제 다운로드 진행
