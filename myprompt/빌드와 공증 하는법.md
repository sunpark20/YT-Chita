# YouTubeDownloader 빌드 & 공증 가이드

## 전체 흐름

```
코드 수정 → 커밋 → 태그 푸시(v1.x.x) → GitHub Actions 자동 빌드/공증 → Release 생성
```

수동으로 할 것은 **태그 푸시**뿐. 나머지는 전부 자동.

---

## 1. 빌드 트리거 방법

```bash
# 태그 생성 후 푸시하면 빌드 자동 시작
git tag v1.2.0
git push origin v1.2.0
```

### 태그 재생성 (빌드 재실행)

```bash
git tag -d v1.1.9                      # 로컬 태그 삭제
git push origin :refs/tags/v1.1.9      # 리모트 태그 삭제
git tag v1.1.9                         # 다시 생성
git push origin v1.1.9                 # 다시 푸시 → 빌드 재시작
```

## 2. 빌드 과정 (자동)

워크플로우 파일: `.github/workflows/build.yml`

### macOS 빌드 (Intel + Apple Silicon 두 개)

| 단계 | 설명 |
|------|------|
| Python 3.11 설치 | `setup-python@v5` |
| 의존성 설치 | `pip install -r requirements.txt` + `pyinstaller` |
| FFmpeg 설치 | `brew install ffmpeg` |
| 인증서 가져오기 | Secrets에 저장된 .p12 인증서를 키체인에 등록 |
| PyInstaller 빌드 | `YouTubeDownloader.spec` 기반으로 .app 생성 |
| 코드 서명 | `codesign` + `entitlements.plist` |
| **공증 (Notarize)** | Apple에 .zip 제출 → 승인 대기 → staple |
| DMG 생성 | `hdiutil`로 .dmg 패키징 |

### 빌드 러너

| 대상 | 러너 | 아키텍처 |
|------|------|----------|
| macOS Intel | `macos-15-intel` (무료) | x86_64 |
| macOS Silicon | `macos-latest` (무료) | arm64 |
| Windows | `windows-latest` | x64 |

## 3. 코드 서명 & 공증 상세

### 코드 서명 (Code Sign)

```bash
codesign --force --options runtime --deep \
  --sign "Developer ID Application" \
  --entitlements entitlements.plist \
  --timestamp \
  "dist/YouTubeDownloader.app"
```

- `--options runtime`: Hardened Runtime 활성화 (공증 필수 조건)
- `--deep`: 내부 프레임워크/바이너리도 서명
- `--entitlements entitlements.plist`: 앱 권한 설정
- `--timestamp`: Apple 타임스탬프 서버 사용

### 공증 (Notarize)

```bash
# 1. zip으로 압축
ditto -c -k --keepParent "dist/YouTubeDownloader.app" dist/YouTubeDownloader.zip

# 2. Apple에 제출 (승인될 때까지 대기)
xcrun notarytool submit dist/YouTubeDownloader.zip \
  --apple-id "$APPLE_ID" \
  --team-id "$APPLE_TEAM_ID" \
  --password "$APPLE_PASSWORD" \
  --wait

# 3. 공증 티켓 스테이플 (오프라인에서도 공증 확인 가능)
xcrun stapler staple "dist/YouTubeDownloader.app"
```

## 4. GitHub Secrets 설정

리포 Settings → Secrets and variables → Actions에서 설정:

| Secret 이름 | 설명 |
|-------------|------|
| `MACOS_CERTIFICATE` | .p12 인증서를 Base64 인코딩한 값 |
| `MACOS_CERTIFICATE_PWD` | .p12 인증서 비밀번호 |
| `APPLE_ID` | Apple 개발자 계정 이메일 |
| `APPLE_TEAM_ID` | Apple 개발자 팀 ID |
| `APPLE_APP_SPECIFIC_PASSWORD` | appleid.apple.com에서 생성한 앱 전용 비밀번호 |

### .p12 인증서 Base64 변환

```bash
base64 -i certificate.p12 | pbcopy
# 클립보드에 복사됨 → GitHub Secret에 붙여넣기
```

## 6. 빌드 실패 시 확인할 것

1. **Actions 탭** 확인: GitHub 리포 → Actions에서 로그 확인
2. **러너 문제**: 인텔은 반드시 `macos-15-intel` 사용 (유료 러너 쓰지 말 것)
3. **인증서 만료**: Apple Developer에서 인증서 유효기간 확인
4. **앱 전용 비밀번호**: appleid.apple.com → 보안 → 앱 전용 암호에서 재발급 가능
