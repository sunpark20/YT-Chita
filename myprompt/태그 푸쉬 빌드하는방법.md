# 태그 푸쉬 및 자동 빌드 방법

새로운 버전을 배포하고 싶을 때는 터미널에서 아래 명령어를 순서대로 입력하세요.

```bash
git add .
git commit -m "update version to 1.1.0 and add sound resources"
git tag v1.1.0
git push origin main --tags
```

### 주의 사항
1. **버전 일치**: 빌드 전에 `src/utils/config.py`와 `YouTubeDownloader.spec`의 버전 번호를 `v1.1.0`과 동일하게 수정했는지 확인하세요.
2. **태그 중복**: 이미 존재하는 태그명(예: v1.1.0)은 다시 사용할 수 없습니다. 다음 배포 시에는 `v1.1.1` 등으로 올려야 합니다.
3. **GitHub Secrets**: 깃허브 레포지토리 설정에 Apple ID 및 인증서 관련 Secrets가 등록되어 있어야 macOS 공증까지 정상 완료됩니다.
